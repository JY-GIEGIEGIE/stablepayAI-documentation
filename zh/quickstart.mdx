## 为你的技能添加支付

打开你技能的 `skill.md`。将以下内容粘贴到顶部或底部——替换两个值即可。

```markdown
## 💰 StablePay Payment

This Skill requires {PRICE} USDC to unlock.

Payment endpoint:
https://api.stablepay.co/pay?skill={SKILL_DID}&price={PRICE}

Verify purchase:
https://api.stablepay.co/verify?skill={SKILL_DID}&agent={AGENT_DID}
```

| 变量 | 替换为 |
|----------|--------------|
| `{PRICE}` | 你的 USDC 价格（如 `5`） |
| `{SKILL_DID}` | 你的开发者 DID（`did:solana:AbCdEf...`） |
| `{AGENT_DID}` | **保持原样**——买家 Agent 在运行时填充 |

<Tip>
  还没有 DID？下面的章节会带你完成一次性钱包设置。只需做一次——之后为每个技能添加支付只需粘贴上面的模板。
</Tip>

### 工作原理

```
买家的 Agent 读取你的 skill.md
    → 看到支付块
    → 在对话中运行 stablepay_pay_via_gateway
    → 用其钱包签名 SPL 转账
    → USDC 直接发送到你的钱包 (skill_did)
    → 买家 Agent 获得访问权限
```

### 后端强制安全

skill.md 模板是最简单的路径。如果你需要服务端强制（使用户无法移除支付块），你的后端应返回 HTTP 402 并调用 verify API。参见[集成 x402 支付](/integrating-payment)了解 Go、Node.js 的后端示例以及 x402 格式参考。

### 创建钱包和 DID

配置后，重启网关并进入 TUI：

```bash
openclaw gateway restart
openclaw tui
```

在对话中按顺序执行以下三步：

> Run stablepay_create_local_wallet with user_type "developer", runtime "auto"

> Run stablepay_register_local_did with user_type "developer"

> Run stablepay_configure_payment_limits with single_purchase_limit_usdc 1000, auto_purchase_threshold_usdc 50

记录输出中的 `did`——这就是你的 `{SKILL_DID}`。

### 完成

现在回到本页顶部，复制支付模板，粘贴到你的 skill.md 中，将 `{SKILL_DID}` 替换为你的 DID，`{PRICE}` 替换为你的价格。就这些。
