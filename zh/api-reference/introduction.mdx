## 基础 URL

```
https://ai.wenfu.cn
```

所有端点均带 `/api/v1/` 前缀。

## 认证

StablePay 使用两种认证方式：

| 方式 | 请求头 | 用途 |
|--------|-----------|----------|
| **DID 签名** | `X-StablePay-DID`、`X-StablePay-Signature`、`X-StablePay-Timestamp`、`X-StablePay-Nonce` | 支付、余额、销售查询 |
| **API 密钥** | `X-API-Key` | 开发者后端验证调用 |

### DID 签名

规范字符串为：

```
{method}\n{path}\n{rawQuery}\n{bodySha256Hex}
```

使用你钱包的 Ed25519 密钥签名。通过 TUI 查询时，插件会自动生成此签名。直接访问 API 时，使用 `stablepay_sign_message` 生成请求头。

### 时间戳与 Nonce

- 时间戳必须在服务器时间的 ±5 分钟内
- Nonce 每个请求必须唯一（防重放）

## 响应格式

所有端点返回统一的信封格式：

```json
{
  "code": 0,
  "message": "success",
  "data": { },
  "request_id": "uuid",
  "timestamp": "2026-05-16T10:30:00Z"
}
```

## 错误码

| 错误码 | 消息 | HTTP | 描述 |
|------|---------|------|-------------|
| 0 | success | 200 | 请求成功 |
| 10001 | invalid parameters | 400 | 参数缺失或格式错误 |
| 10002 | resource not found | 404 | 请求的资源不存在 |
| 10003 | permission denied | 403 | 权限不足 |
| 10004 | signature verification failed | 401 | DID 签名无效或已过期 |
| 20001 | insufficient balance | 402 | 钱包余额不足 |
| 20002 | payment already exists | 409 | 重复支付（幂等性） |
| 20003 | blockchain network error | 503 | Solana 网络问题 |
| 20004 | gas subsidy failed | 500 | 热钱包 Gas 费失败 |
| 30001 | internal server error | 500 | 服务端意外错误 |
| 30002 | service unavailable | 503 | 依赖服务宕机 |
| 30004 | rate limit exceeded | 429 | 请求过于频繁 |

## HTTP 402 约定

代表付费资源的端点在调用者尚未购买访问权限时返回 **HTTP 402 Payment Required**。响应 body 包含 [x402 标准格式](/x402-format)或[旧版格式](/x402-format#legacy-format-backward-compatible)的支付挑战。

## 货币精度

- 外部 API：`amount` 以**十进制字符串**表示（如 `"5.00"`）
- 内部/链上：**整数最小单位**，6 位小数（`5000000` = 5.00）
- 支持的货币：`USDC`、`USDT`
