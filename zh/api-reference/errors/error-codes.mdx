## HTTP 状态码语义

并非每个非 200 响应都是错误。StablePay 使用 HTTP 状态码来信号不同情况：

| 状态 | 含义 | 是错误吗？ | 操作 |
|--------|---------|-------------------|--------|
| **200** | 成功 | 否 | 查看 JSON body 中的 `code`。`code: 0` = 真正成功；`code ≠ 0` = 业务拒绝 |
| **400** | 请求错误 | 是——你的请求格式有误 | 对照端点文档检查参数，修复后重试 |
| **401** | 未授权 | 是——DID 签名缺失或无效 | 重新生成 timestamp + nonce，重新签名规范字符串，重试 |
| **402** | 需要支付 | **否**——这是正常业务流程，非系统错误 | 从 body 提取支付挑战，构建并提交签名交易 |
| **403** | 禁止 | 是——你无权访问此资源 | 确认你使用了正确的 DID 或 API Key |
| **404** | 未找到 | 是——资源不存在 | 验证 DID 或 tx_id 正确且已注册 |
| **409** | 冲突 | 是——重复或幂等冲突 | 检查 `X-Idempotency-Key`；409 在支付时表示已成功 |
| **429** | 限流 | 是——你发送的请求过多 | 等待并使用退避重试。参见[重试与幂等性](/api-reference/errors/retry-idempotency) |
| **500** | 内部服务错误 | 是——服务端出错了 | 使用指数退避重试。如持续出现则报告 |
| **503** | 服务不可用 | 是——依赖项（Solana RPC、数据库）宕机 | 使用退避重试。Solana 拥堵是最常见的原因 |

<Info>
  **402 不是 bug。** 如果你的后端从 `GET /api/v1/pay/require` 收到 HTTP 402，这是 Agent 尚未购买技能时的预期响应。它是支付流程的触发器，而非失败。
</Info>

### 哪些状态码可以重试？

| 重试安全 | 状态码 | 策略 |
|------------|-------------|----------|
| 是 | `429`、`500`、`503` | 指数退避：1s → 2s → 4s → ... 最多 3 次尝试 |
| 是 | `401` | 刷新 timestamp + nonce，重新签名，重试一次 |
| 否 | `400`、`403`、`404` | 先修复根本原因再重试 |
| 特殊情况 | `409` | `POST /api/v1/pay` 时：视为成功（支付已存在）。其他端点：修复幂等键 |
| 特殊情况 | `402` | 不是失败——继续支付流程 |

## 业务错误码

每个响应 body（包括错误）都包含一个 `code` 字段。`0` = 成功。所有其他码表示业务级拒绝。

### 1xxxx — 客户端错误

| 错误码 | 消息 | HTTP | 原因 | 恢复方式 |
|------|---------|------|-------|----------|
| `10001` | invalid parameters | 400 | 缺少必需字段、类型错误或值超出范围 | 对照端点参数表检查并使用修正值重试 |
| `10002` | resource not found | 404 | DID、tx_id 或 skill_did 不存在或未注册 | 验证标识符正确。对于 DID，先调用 `POST /api/v1/did/register` |
| `10003` | permission denied | 403 | 资源对应的 DID 错误，或 API Key 权限不足 | 使用拥有该资源的 DID。对于验证端点，确认你的 API Key |
| `10004` | signature verification failed | 401 | 时间戳超出 ±5 分钟窗口、nonce 重复使用或 Ed25519 签名与 DID 公钥不匹配 | 刷新时间戳到当前时间，生成新的 nonce，重新签名规范字符串 |

### 2xxxx — 支付错误

| 错误码 | 消息 | HTTP | 原因 | 恢复方式 |
|------|---------|------|-------|----------|
| `20001` | insufficient balance | 402 | Agent 钱包 USDC 余额不足以支付 | 用户需向钱包存入 USDC。告诉他们钱包地址和差额 |
| `20002` | payment already exists | 409 | 同一 `agent_did + skill_did` 对已有完成支付 | **视为成功。** 购买记录已存在。继续提供资源 |
| `20003` | blockchain network error | 503 | Solana RPC 不可达或交易被网络拒绝 | 最多重试 3 次，使用指数退避（2s → 4s → 8s）。如全部失败，告知用户稍后再试 |
| `20004` | gas subsidy failed | 500 | 服务端热钱包无法为此交易覆盖 SOL Gas 费 | 5 秒后重试一次。如持续出现则向平台运营者报告 |

### 3xxxx — 服务端错误

| 错误码 | 消息 | HTTP | 原因 | 恢复方式 |
|------|---------|------|-------|----------|
| `30001` | internal server error | 500 | 后端服务中未处理的异常 | 使用指数退避重试。如多次尝试持续出现则向平台团队报告 |
| `30002` | service unavailable | 503 | 依赖服务（MySQL、Redis、RocketMQ）不可达 | 使用退避重试。平台健康检查应自动恢复此问题 |
| `30003` | database connection error | 503 | MySQL 连接池耗尽或实例不可达 | 使用退避重试。如持续出现，平台团队可能已在响应事件 |
| `30004` | rate limit exceeded | 429 | 超出每个 DID 或每个 IP 的请求限制 | 等待至少 60 秒后重试。降低请求频率。参见[重试与幂等性](/api-reference/errors/retry-idempotency) |

## 速率限制

| 范围 | 限制 | 窗口 |
|-------|-------|--------|
| IP 地址 | 100 请求 | 每分钟 |
| DID | 50 请求 | 每分钟 |
| 支付端点（`POST /api/v1/pay`） | 10 请求 | 每分钟 |

触发限制时，API 返回 `30004` 及 HTTP `429`。响应在可用时包含 `Retry-After` 头。降低调用频率——以相同速率立即重试会持续触发限制。
