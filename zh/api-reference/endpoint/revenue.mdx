## 用途

查询开发者技能的汇总收益统计——总收入、总销售数量和每日趋势。

## 收益 vs 销售

| 端点 | 返回 | 最适合 |
|----------|---------|----------|
| `GET /api/v1/revenue` | 汇总摘要 + 每日趋势 | 仪表盘、收入概览 |
| `GET /api/v1/sales` | 单个交易记录 | 交易级明细、对账 |

使用 `revenue` 获取高层指标，使用 `sales` 深入查看单笔购买。

## 认证

需要 **DID 签名认证**——你必须使用拥有 `skill_did` 的钱包进行签名。

## 查询参数

| 参数 | 必需 | 类型 | 描述 |
|-----------|----------|------|-------------|
| `skill_did` | 是 | string | 要查询的开发者/技能 DID |

## 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "total_revenue": "120.00",
    "total_sales": 24,
    "currency": "USDC",
    "sales_trend": [
      {"date": "2026-05-28", "amount": "15.00"},
      {"date": "2026-05-29", "amount": "25.00"}
    ]
  }
}
```

| 字段 | 描述 |
|-------|-------------|
| `total_revenue` | 历史总收入（十进制 USDC） |
| `total_sales` | 购买总次数 |
| `currency` | 结算货币 |
| `sales_trend` | 每日明细。时间范围和粒度取决于 Query Service 实现。 |

## 多技能

如果你在不同的 DID 下发布了多个技能，需要为每个 `skill_did` 分别调用此端点。没有"汇总所有技能"的聚合端点——分别查询每个技能。
