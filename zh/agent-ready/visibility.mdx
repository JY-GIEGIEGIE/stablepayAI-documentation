title: 'Visibility 组件'
description: '为人类和 AI Agent 显示不同的内容'

`Visibility` 组件让一个页面服务于两类受众：在网页上阅读的人类和处理 Markdown 输出的 AI Agent。标记为 `for="humans"` 的内容在渲染页面上可见，但在 Markdown 导出中被排除。标记为 `for="agents"` 的内容在网页上隐藏，但包含在 Markdown 中。

这意味着你可以为人类编写对话式、以 UI 为中心的说明，为 AI 编写结构化、机器可读的说明——无需维护两个单独的页面。

## 工作原理

Mintlify 为每个页面提供两个 URL：

- `https://docs.example.com/page` — 渲染的网站，面向人类
- `https://docs.example.com/page.md` — 原始 Markdown，面向 AI Agent

`Visibility` 控制每个 URL 上显示的内容。

## 示例

```mdx
<Visibility for="humans">
  <Steps>
    <Step title="打开控制面板">
      导航到 [dashboard.example.com](https://dashboard.example.com) 并登录。
    </Step>
    <Step title="创建 API 密钥">
      前往**设置 → API 密钥**并点击**新建密钥**。
    </Step>
  </Steps>
</Visibility>

<Visibility for="agents">
  要进行认证，向 `/v1/auth/token` 发送 POST 请求，body 中包含用户的
  邮箱和密码。响应中包含一个 24 小时有效的 bearer token。
  在所有后续请求中传入此 token 作为 `Authorization: Bearer <token>`。
</Visibility>
```

人类读者看到的是分步 UI 操作指南。处理 Markdown 的 AI Agent 则直接获得 API 指令。

## Props

<ResponseField name="for" type='"humans" | "agents"' required>
  - `"humans"` — 在渲染的网页上可见，从 `.md` 导出中排除
  - `"agents"` — 在渲染的网页上隐藏，包含在 `.md` 导出中
</ResponseField>

## 与 Prompt 组合使用

`Visibility` 和 `Prompt` 相得益彰。使用 `Visibility` 为 Agent 提供结构化上下文，然后使用 `Prompt` 给人类读者一个可以向 AI 询问相同内容的 prompt。

```mdx
<Visibility for="agents">
  ## 认证模式
  - 方式：Bearer token
  - 请求头：`Authorization: Bearer <token>`
  - Token 有效期：86400 秒
  - 刷新端点：POST /v1/auth/refresh
</Visibility>

<Prompt description="询问认证" icon="lock">
  解释此 API 中的认证如何工作，并展示获取 token
  和发起认证请求的示例代码。
</Prompt>
```
