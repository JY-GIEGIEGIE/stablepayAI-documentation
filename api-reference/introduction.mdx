## Base URL

```
https://ai.wenfu.cn
```

All endpoints are prefixed with `/api/v1/`.

## Authentication

StablePay uses two authentication methods:

| Method | Header(s) | Used for |
|--------|-----------|----------|
| **DID Signature** | `X-StablePay-DID`, `X-StablePay-Signature`, `X-StablePay-Timestamp`, `X-StablePay-Nonce` | Payment, balance, sales queries |
| **API Key** | `X-API-Key` | Developer backend verification calls |

### DID Signature

The canonical string is:

```
{method}\n{path}\n{rawQuery}\n{bodySha256Hex}
```

Signed with your wallet's Ed25519 key. The plugin generates this automatically for TUI queries. For direct API access, use `stablepay_sign_message` to generate headers.

### Timestamp & Nonce

- Timestamp must be within ±5 minutes of server time
- Nonce must be unique per request (prevents replay)

## Response format

All endpoints return a unified envelope:

```json
{
  "code": 0,
  "message": "success",
  "data": { },
  "request_id": "uuid",
  "timestamp": "2026-05-16T10:30:00Z"
}
```

## Error codes

| Code | Message | HTTP | Description |
|------|---------|------|-------------|
| 0 | success | 200 | Request succeeded |
| 10001 | invalid parameters | 400 | Missing or malformed parameters |
| 10002 | resource not found | 404 | Requested resource does not exist |
| 10003 | permission denied | 403 | Insufficient permissions |
| 10004 | signature verification failed | 401 | DID signature invalid or expired |
| 20001 | insufficient balance | 402 | Wallet balance too low |
| 20002 | payment already exists | 409 | Duplicate payment (idempotency) |
| 20003 | blockchain network error | 503 | Solana network issue |
| 20004 | gas subsidy failed | 500 | Hotwallet gas fee failure |
| 30001 | internal server error | 500 | Unexpected server error |
| 30002 | service unavailable | 503 | Dependency service down |
| 30004 | rate limit exceeded | 429 | Too many requests |

## HTTP 402 convention

Endpoints that represent paid resources return **HTTP 402 Payment Required** when the caller hasn't purchased access. The response body contains either an [x402 standard format](/x402-format) or [legacy format](/x402-format#legacy-format-backward-compatible) payment challenge.

## Currency precision

- External API: `amount` in **string decimal** (e.g., `"5.00"`)
- Internal/on-chain: **integer minor units** with 6 decimals (`5000000` = 5.00)
- Supported currencies: `USDC`, `USDT`
