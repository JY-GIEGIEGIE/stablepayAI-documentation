## Querying revenue

Once your paid skill is live, you can monitor earnings through OpenClaw conversation or via the REST API.

### via TUI conversation

> Run stablepay_query_sales with skill_did "did:solana:`<your-did>`"

This calls `GET /api/v1/sales?skill_did=...` with your wallet's DID signature for authentication.

### via REST API

```bash
curl -H "X-StablePay-DID: did:solana:<your-did>" \
     -H "X-StablePay-Signature: <signature>" \
     -H "X-StablePay-Timestamp: <ISO-timestamp>" \
     -H "X-StablePay-Nonce: <nonce>" \
     "https://ai.wenfu.cn/api/v1/sales?skill_did=did:solana:<your-did>"
```

<Info>
  Sales/balance queries require DID signature authentication. The plugin generates these headers automatically when you query via TUI tools. For direct API access, use `stablepay_sign_message` to generate the required headers.
</Info>

## Available endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/v1/sales?skill_did=...` | GET | Sales records for a specific skill |
| `/api/v1/revenue?skill_did=...` | GET | Revenue summary (total + trends) |
| `/api/v1/balance?agent_did=...` | GET | Wallet balance + monthly spending |
| `/api/v1/transactions?did=...` | GET | Transaction history (purchase/revenue) |

## Settlement timing

Payments settle directly on Solana. USDC is **yours** as soon as the chain confirms the transfer — typically 5-30 seconds after the buyer submits payment.

When it becomes **visible in queries** depends on the async pipeline:

```
Buyer Agent pays
  → Solana transaction submitted (~5s)
    → Chain confirmed — USDC is in your wallet now
      → RocketMQ event → Verification Service (async, normally <5s)
        → Purchase record written to MySQL
          → Visible in sales/revenue API queries
```

<Info>
  **Chain confirmation = your money. API visibility = your dashboard.** The two happen at slightly different times. For independent verification of any transaction, use [Solana Explorer](/settlement#verify-on-solana-explorer) with the `tx_hash`. For full reconciliation guidance, see [Settlement & reconciliation](/settlement).
</Info>

## Tracking multiple skills

If you publish multiple skills, you can use different `skill_did` values for each (by creating separate developer wallets), or use the same DID across all skills and differentiate by metadata. For the trade-offs between these two strategies, see [Multi-skill DID strategy](/settlement#multi-skill-did-strategy).

Query each skill separately:

> Run stablepay_query_sales with skill_did "did:solana:`<skill-1>`"
> Run stablepay_query_sales with skill_did "did:solana:`<skill-2>`"
